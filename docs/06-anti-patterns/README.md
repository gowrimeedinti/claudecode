# Section 6: Anti-Patterns

_Common mistakes teams make with Claude Code and how to avoid them._

## Topics to cover

- Giving too little context and getting generic output
- Letting Claude make too many changes without reviewing
- Treating Claude's output as ground truth without testing
- Not writing a CLAUDE.md (Claude doesn't know your conventions)
- Pasting entire large files when a summary would do
- Using Claude Code for tasks requiring live data / prod access without guardrails
- Over-relying on Claude for architecture decisions
